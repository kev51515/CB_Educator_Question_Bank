Three identities for the same product

Same screens, same data, three different design systems. Each sample shows the educator dashboard, the Modules surface, the student "what's next" view, the Bluebook-calm test runner, and a dark-mode inset — built on full token sheets so the winner can be lifted straight into tailwind.config.js.

Open each in its own tab. Judge at desktop width. The test runner stays light and calm in all three — that's deliberate.

Direction A

Scholar's Desk

Warm paper, ink, and serif type. The page is never white. Calm enough that the LMS and the test runner feel like one serene world.

How to read these

All three keep what already works: the Canvas information architecture, the Linear-style rail + ⌘K, the three-domain accent concept (academic / counseling / coaching), and a Bluebook-neutral test runner so practice rehearses test day. What changes is the identity layer: ground color, type voices, depth system, and how progress is visualized.

Full research findings, the current-UI audit, and the migration plan are in PROPOSAL.md in this folder. The directions are mixable — e.g. the "Instrument" chassis with "Scholar's Desk" serif passages is a legitimate pick.

Chosen: Ivy Ledger. Six real app screens rebuilt on the refined kit live in ivy-ledger/ — dashboard, modules, gradebook, test overview, student home, and the test runner.
